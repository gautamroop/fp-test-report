🐛 Issues & Findings

🔴 Critical: SKU Filter Not Working

Test Case: TC_CE3398_008 (ETM-5797)

Severity: High

Description: When providing a non-existent SKU parameter, the API returns all products instead of an empty result set.

Request Example ▼
GET https://api.fisherpaykel.com/uat/v6/products?channel=mobile&market=NZ&SKU=INVALID_SKU_DOES_NOT_EXIST

Expected Behavior: Return 200 OK with empty array []

Actual Behavior: Returns 200 OK with 8 products (SKU parameter is ignored)

Impact:

  • High - Core filtering functionality not working
  • Affects Acceptance Criteria Scenario 7
  • May also affect Scenario 6 (valid SKU filtering)

Recommendation: Create defect ticket and verify with development team before production release.
